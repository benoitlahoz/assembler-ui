# Usage

Once you have AssemblerUI set up, you can start using components in your project. This guide will show you how to add components, customize them, and follow best practices.

## Adding a Component

AssemblerUI follows a **copy & paste** approach. Here's how to add a component to your project:

::steps

### Browse the Component Library

Visit the [Components page](/components) to browse all available components. Each component includes:

- Live preview
- Complete source code
- Props documentation
- Usage examples

### Copy the Component Code

Click on the component you want to use and copy the source code. For example, the Button component:

```vue [components/ui/Button.vue]
<template>
  <component
    :is="asChild ? 'slot' : 'button'"
    :class="cn(buttonVariants({ variant, size }), $attrs.class ?? '')"
    :disabled="disabled"
    v-bind="$attrs"
  >
    <slot />
  </component>
</template>

<script setup lang="ts">
import { buttonVariants, type ButtonVariant, type ButtonSize } from '.';
import { cn } from '@/lib/utils';

defineOptions({
  name: 'Button',
  inheritAttrs: false,
});

export interface ButtonProps {
  variant?: ButtonVariant;
  size?: ButtonSize;
  asChild?: boolean;
  disabled?: boolean;
}

withDefaults(defineProps<ButtonProps>(), {
  variant: 'default',
  size: 'default',
  asChild: false,
  disabled: false,
});
</script>
```

### Paste and Customize

1. Create the component file in your project
2. Paste the code
3. Customize as needed
4. Import and use in your components

::

## Component Structure

AssemblerUI components follow a consistent structure:

### File Organization

```
components/
└── ui/
    └── button/
        ├── Button.vue       # Main component
        ├── index.ts         # Exports and types
        └── variants.ts      # Styling variants (if needed)
```

### Component Anatomy

Each component typically includes:

- **Template**: The HTML structure with proper accessibility
- **Props Interface**: TypeScript definitions for component props
- **Styling**: Class variants using `class-variance-authority`
- **Composables**: Vue 3 Composition API features when needed

## Customization

### Styling Variants

Components use `class-variance-authority` for styling variants:

```ts [components/ui/button/index.ts]
import { cva, type VariantProps } from 'class-variance-authority'

export const buttonVariants = cva(
  // Base styles
  "inline-flex items-center justify-center rounded-md text-sm font-medium transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:opacity-50 disabled:pointer-events-none ring-offset-background",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        destructive: "bg-destructive text-destructive-foreground hover:bg-destructive/90",
        outline: "border border-input hover:bg-accent hover:text-accent-foreground",
        secondary: "bg-secondary text-secondary-foreground hover:bg-secondary/80",
        ghost: "hover:bg-accent hover:text-accent-foreground",
        link: "underline-offset-4 hover:underline text-primary",
      },
      size: {
        default: "h-10 py-2 px-4",
        sm: "h-9 px-3 rounded-md",
        lg: "h-11 px-8 rounded-md",
        icon: "h-10 w-10",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
)

export type ButtonVariant = VariantProps<typeof buttonVariants>['variant']
export type ButtonSize = VariantProps<typeof buttonVariants>['size']
```

### CSS Custom Properties

Customize colors by modifying CSS variables:

```css [assets/css/main.css]
:root {
  --primary: 220 100% 50%;     /* Custom primary color */
  --primary-foreground: 0 0% 100%;
  --radius: 0.75rem;           /* Custom border radius */
}
```

### Component Props

Pass props to customize component behavior:

```vue [pages/example.vue]
<template>
  <div>
    <!-- Different variants -->
    <Button variant="default">Default</Button>
    <Button variant="secondary">Secondary</Button>
    <Button variant="outline">Outline</Button>
    
    <!-- Different sizes -->
    <Button size="sm">Small</Button>
    <Button size="default">Default</Button>
    <Button size="lg">Large</Button>
    
    <!-- Custom styling -->
    <Button class="bg-green-500 hover:bg-green-600">
      Custom Color
    </Button>
  </div>
</template>
```

## Best Practices

### File Organization

Keep your components organized:

```
components/
├── ui/           # AssemblerUI components
│   ├── button/
│   ├── card/
│   └── input/
├── layout/       # Layout components
└── feature/      # Feature-specific components
```

### Import Strategy

Use consistent import patterns:

```vue
<script setup lang="ts">
// Import components
import { Button } from '@/components/ui/button'
import { Card, CardContent, CardHeader } from '@/components/ui/card'

// Import utilities
import { cn } from '@/lib/utils'
</script>
```

### Accessibility

AssemblerUI components include accessibility features by default:

- Proper ARIA attributes
- Keyboard navigation
- Screen reader support
- Focus management

Always maintain these features when customizing components.

### TypeScript Support

Take advantage of full TypeScript support:

```vue
<script setup lang="ts">
import type { ButtonProps } from '@/components/ui/button'

// Type-safe props
const buttonProps: ButtonProps = {
  variant: 'outline',
  size: 'lg',
  disabled: false
}
</script>
```

## Common Patterns

### Composite Components

Build complex UIs by combining simple components:

```vue [components/UserCard.vue]
<template>
  <Card class="w-96">
    <CardHeader>
      <CardTitle>{{ user.name }}</CardTitle>
      <CardDescription>{{ user.email }}</CardDescription>
    </CardHeader>
    <CardContent>
      <div class="flex space-x-2">
        <Button variant="default">
          Follow
        </Button>
        <Button variant="outline">
          Message
        </Button>
      </div>
    </CardContent>
  </Card>
</template>
```

### Wrapper Components

Create wrapper components for common use cases:

```vue [components/SubmitButton.vue]
<template>
  <Button
    type="submit"
    :disabled="loading"
    v-bind="$attrs"
  >
    <Loader2 v-if="loading" class="mr-2 h-4 w-4 animate-spin" />
    <slot />
  </Button>
</template>

<script setup lang="ts">
import { Button } from '@/components/ui/button'
import { Loader2 } from 'lucide-vue-next'

interface Props {
  loading?: boolean
}

withDefaults(defineProps<Props>(), {
  loading: false
})
</script>
```
